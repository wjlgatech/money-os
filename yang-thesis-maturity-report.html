7 Gaps to Close

1. Thesis Storage + CRUD
Need: theses table: id, name, source, categories (JSON), status, created_at
Need: thesis_tickers join: thesis_id → ticker, category, target_pct, conviction
Need: /api/theses GET/POST/PUT/DELETE endpoints
Effort: ~2 hours. Schema + migration + REST routes. Straightforward Drizzle work.
2. Thesis-Aware Auto-Trader
Today: daily-auto-trader.ts picks top 5 ENTRY zone tickers regardless of thesis
Need: Filter proposals by active thesis. Prioritize under-allocated categories. Respect per-category target %.
Example: Yang says 15% commodities. Portfolio has 2.8% gold. Auto-trader should prioritize COPX/SLV/GDX over more semis.
Effort: ~3 hours. Modify proposal generation to read thesis allocations + current exposure.
3. Partial Position Exits
Today: paperTrader.checkExits() sells 100% of position on stop/TP hit
Need: executeSell(ticker, shares, price, reason) already supports partial! But checkExits always passes pos.shares
Fix: Add partialExitPct to position config. When TP hit, sell that % only. Update remaining stop to breakeven.
Effort: ~1 hour. The hard part (partial sell) is already built. Just needs the trigger logic.
4. Contrarian Exit Signal (VIX + RSI Proxy)
Yang says: "Sell 20-30% when news turns positive"
Quantifiable proxy: VIX drops below 15 AND basket RSI > 70 AND thesis P&L > +15%
Today: VIX endpoint exists (/api/vix). RSI computed per ticker. No basket-level aggregation.
Need: Thesis-level composite signal: weighted RSI across thesis tickers + VIX threshold + thesis P&L threshold
Effort: ~2 hours. Aggregate existing per-ticker signals into a thesis-level metric.
5. Thesis-Level Monitoring + Rebalancing
Need: Daily check: "Is my semiconductor allocation at 45%? Is commodities at 15%?"
Need: Auto-generate rebalance proposals when drift exceeds threshold (e.g., >3% off target)
Effort: ~3 hours. Compare thesis targets vs actual holdings, generate trimming/adding proposals.
6. thesis-to-trades Skill → Persistent
Today: skills/thesis-to-trades/SKILL.md does a one-shot analysis. Results disappear after the conversation.
Need: Save thesis to DB via /api/theses. Claude skill becomes the UX for CRUD.
Effort: ~1 hour. Wire existing skill output to new API endpoints.
7. Holdings Sync (Profile → Live)
Today: profile/holdings.md is a manual screenshot import (last updated 3/24)
Need: Alpaca API already provides live positions. But Fidelity (actual broker) has no API.
Workaround: Periodic screenshot → Claude vision OCR → update holdings.md. Already works but manual.
Real fix: Plaid integration (built but needs API key) would auto-sync.

What the Backtest Can't Do (Yet)

Can't backtest Yang's THESIS — only individual tickers
backtestPortfolio() runs N independent single-ticker backtests and sums P&L.
Yang's thesis requires: coordinated entry (buy the basket when Hormuz signal fires), allocation-weighted sizing (40% semis, 15% commodities), and basket-level exits (sell 25% of everything when VIX < 15).
The backtest engine treats each ticker independently — no shared capital pool with allocation constraints.
No macro event backtesting
Yang's entry signal is geopolitical (Hormuz Strait reopening). The backtest engine uses technical signals (RSI, MACD, trendline proximity). There's no way to say "on date X, the macro trigger fired — enter all thesis positions."
Workaround: Manually set the entry date and test the subsequent performance. The engine supports this with a fixed start date.
No benchmark comparison built in
CLAUDE.md says best config is "Trend50 + 2ATR + 8%TP — doesn't beat SPY buy-and-hold yet."
The backtest doesn't auto-compare against SPY. You have to manually compute SPY return for the same period.
Yang's thesis should be benchmarked against: SPY buy-and-hold, 60/40 portfolio, and your current portfolio's return.

Bottom Line

Money OS is a great per-ticker trading system wrapped in a not-yet-built thesis orchestration layer. The plumbing — data pipeline, indicators, scanner, signals, position sizing, trade gate, paper/live execution — is genuinely well-built. What's missing is the portfolio intelligence layer: the ability to think in baskets, allocations, and coordinated exits. That's the difference between "a stock screener with a trade button" and "an investment co-pilot."

The good news: the hardest engineering (trendline computation, backtest walk-forward, broker integration) is done. The thesis layer is mostly glue code + a new table + modified proposal logic. ~12 hours of focused work.
