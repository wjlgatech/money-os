# Product Principles

## User value

- Start with clarity before optimization.
- Explain what matters now in the user's own financial context.
- Help users avoid unforced errors before trying to create upside.

## Trust

- Separate facts, inferences, and recommendations.
- Show confidence and uncertainty explicitly.
- Never hide degraded data or stale assumptions.

## Automation

- Treat automation as earned trust, not a default feature.
- Require visible policies, limits, and approval boundaries.
- Prefer reversible and simulated actions before production execution.

## Product discipline

- Solve real financial pain before adding prestige features.
- Favor repeatable workflows over impressive but fragile demos.
- Convert expert thinking into operational systems, not personality mimicry.
